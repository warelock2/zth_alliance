Cipher Bridge

In order for the cold storage wallet device to work, it needs to be tethered via USB to a host system running Pirate Chain hot wallet software, such as Treasure Chest. Currently, we have a PC desktop Cipher Bridge program. The Pirate Chain software developers are currently rewriting their lite wallet software and plan to include the Cipher Bridge USB communications and "Send" tab function.

Beta Testing

On 2025-11-25, Heiko of the Pirate Chain Discord community, graciously offered his time and effort to run the Cipher Wallet Hardware Wallet device through its paces. Here is a direct quote from Heiko on his experience using the device:

"The device is very compact, unobtrusive, easy to pack with my other electronics in my laptop bag. Using it is very straightforward, the onscreen menus are easy to navigate. It interacts very well with Treasure Chest, transacting with it is quick and simple for both sending and receiving. It is easier and quicker to use than other hardware wallets I have used in the past." -- Heiko

Here is a list of activities I observed him doing during the beta testing session:

  1. Physically handling the device, plugging it in via USB, and observing it boot up
  2. Interacting with the device physically to set a PIN code (I looked away while he entered in the actual PIN code)
  3. Going through all of the menus on the device to view the newly generated Pirate Chain cold storage address, scanning the on-device, on-screen QR code with the address embedded in it, viewing his seed phrase (I also looked away for this part), going into the settings menu, taking a quick view of the "Recover from seed phrase" with its on-device alpha keyboard
  4. Installing the Cipher Bridge ".deb" archive file on his Linux laptop, then running it to open the GUI Cipher Bridge window
  5. Adding the "dialout" group to his Linux non-root user's account and rebooting for that to take effect in his desktop session
  6. Interacting with the various Cipher Bridge "Advanced" tab features to upgrade the device's operating system over the USB cable, check for updates of the on-device "pirated" headless, non-GUI Treasure Chest software, importing of his newly generated cold storage wallet's viewing key into his desktop GUI Treasure Chest instantly with no forced blockchain rescan (which is fine, since the cold storage address was brand new and had no on-blockchain transactions needing scanning to build the cold storage device's balance), interacting with the various popup screens caused by the Cipher Bridge-generated requests like the device OS upgrade
  7. Sending some Pirate Chain from one of his existing addresses to the cold storage wallet via his desktop GUI Treasure Chest software
  8. Sending the Pirate Chain from his cold storage wallet back to his hot wallet address using the desktop GUI Cipher Bridge software to prepare and send the transaction signing request to the device, each time interacting with the accept/reject transaction confirmation popup screen on the device, visually verifying the details on the on-device popup message, then clicking the on-device "Accept" button, then watching the on-device signing progress bar, then seeing the simultaneous desktop transaction popup from Treasure Chest showing the same details as what the on-device message said, observing the updated cold storage address balance in Treasure Chest
  9. Powering the device down using the on-device, on-screen "Power" button, then waiting for the device to show the shutdown screen, then unplugging the device from USB
